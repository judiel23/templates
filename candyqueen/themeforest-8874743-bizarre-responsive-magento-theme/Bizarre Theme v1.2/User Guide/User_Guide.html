Demo store's installation

In this theme package we included sample data with settings of demo version. Follow the instructions below in order to install exactly the same store as on demo version:

  1. Before magento installation create empty database using phpMyAdmin or similar tools
  2. Import Sources/demo/dump.sql.gz into database that was created in previous step
  3. Install demo magento store:
    1. Go to magento download page choose the recent version of magento store and download it
    2. Upload and unzip downloaded file with magento to the folder on your server where magento should be installed
    3. Open browser and proceed to the folder where magento was unzipped and load default Magento setup wizard
    4. Follow the wizard's instructions and finish magento installation
  4. When the demo store with demo sample data is installed follow successively the steps that were described in sections: "Prepare your magento", "Upload the theme files", "Install the theme"

Prepare your magento

1. Refresh and Disable all the Magento cache

When configuring your store, whatever theme or anything else you should refresh all the Magento cache and then completely disable it. To get it done go to System -> Cache Management

2. Reindex magento data

After making any changes to magento data (products, categories) you should reindex the data. To get it done go to System -> Index Management

3. Backup Magento files and the store database

It saves lot's of your time if something goes wrong. To create backup go to System-> Tools-> Backups

4. Disable compilation mode

To get it done go to System -> Tools -> Compilation

5. Disable Merge JavaScript and CSS Files

To get it done go to System -> Configuration -> Developer

6. Log out from Magento admin panel

It must be done to refresh the magento access control system

Upload the theme files

  1. If your FTP manager allows to unzip files then open your downloaded theme package, navigate to Bizarre Theme vX.X/Theme Files and using your FTP client upload .zip file Theme_vX.X.zip to the root directory of your magento
  2. Unzip Theme_v.X.X.zip
  3. If your FTP client doesn't allow to unzip files on server then upload and unzip theme .zip files using CPanel:

    4. Despite the fact that the .zip file contains the same folders as magento none of the magento core file will be overridden. It was made with the aim to make upload process easier
    • Bizarre theme was developed based on magento 1.9.x.x version. If you are running store on that magento version, you have just upload folders from the Theme folder
    • If you are running store on magento 1.7.x.x after uploading of theme files, open theme package navigate to Theme Files -> Patch for magento 1.7.x.x. and move founded folder to the root of your magento
    • If you are running store on magento 1.8.0.0 after uploading of theme files, open theme package navigate to Theme Files -> Patch for magento 1.8.0.0. and move founded folder to the root of your magento
    • If you are running store on magento 1.8.1.0 after uploading of theme files, open theme package navigate to Theme Files -> Patch for magento 1.8.1.0. and move founded folder to the root of your magento
  5. Set correct file permissions for uploaded folders. File permissions depend on server settings. If you have any doubts about file permissions, please sent request to your hosting provider. More info about file permissions you can find here Magento WIKI: Filesystem permission
  6. Log in to magento admin panel

Install the theme

There are two ways how you can get the theme installed and configured

Using "1 Click Installation" feature

This feature allows you to get your store installed and configured in a few clicks but we do not recommend using it on live stores because you can lose content of your homepage or/and static blocks. This feature is provided only for testing the theme on a test server. To install theme using this feature go to Meigee -> Bizarre -> Activate Theme. In that menu you can mark 2 checkbox in order all static blocks and pages were created automatically and choose store for which theme should be applied. Be aware that if you already created some static blocks or home page with your own content they will be replaced with theme static content in case ID of your blocks and page will match predefined static blocks and pages. After you configured installation page you just need to activate theme by clicking on the activation button: the cache will be cleared and you will be logout automatically.

Manual Installation

This is the only right way of theme's installation if you prefer to install it on live store

  1. Go to System -> Configuration -> Design page of your admin panel and set the field "Current Package Name" as bizarre. Click Save Config button
  2. Import static blocks and pages
  3. Upload media folder from Sources/demo images to the root of your magento. Please notice that real images from preview are not included in the package due to licensing restrictions so are replaced with blank ones

Known problems with installation

If you get any problems with theme's installation first of all make sure that you have followed all steps indicated in "Prepare your magento" section.

404 Not Found when trying to open theme configuration page
Solution: It's pretty common issue. Just clear all the magento cache then logout from the admin panel and login again

"Package with this name does not exist and cannot be set" error when setting the package name
Solution 1. Make sure you have uploaded the theme files in the correct way. Note that folders structure of uploaded files must match to the structure of files in downloaded package.
Solution 2. Check if you have set correct file permissions

Theme Options



If theme installed correctly you will see free-standing module with several options in admin panel. Let's take a look at every menu option of the module.

General options menu consist of next sections:

If you choose here for example Theme Design - skin1, but will work in Theme Design - skin2 vertical tab, the changes won't be saved. More detailed about subtheme configuration you will read in section Pre-defined subthemes of this documentation

  • Retina Ready that submenu enables retina ready feature. Be aware that if Retina Ready option is enabled you have to upload your retina ready images for the images which are used in static content and indicate the path for them (i.e. data-srcX2="{{media url="your_media_folder/your_retina_ready_image_img@x2.jpg"}}) otherwise your clients will see nothing on their mobile devices with retina displays. Retina version of images are created automatically for the product images, so no need to upload different version of product image - just upload the image with the highest resolution

  • Layout here you can choose wide or boxed website layout, enable responsiveness of your e-commerce store and define style of my cart page (accordion or simple list). More detailed about shopping cart you can read in section Shopping cart page of current documentation

  • Header this menu allows you to activate sticky header. Header menu also allows you to add or remove such elements as cart, search box, small store logo and navigation menu from the sticky header. Same here you can decide on which devices sticky header will display those elements: you can show them on all devices or only on desktop. Keep in mind that if the navigation menu has many top level categories, other elements will have lack of space. Under the header submenu you also can choose background for header. If "Part of 1st banner" is set then the 1st banner will be underneath the header. Colors won't be changed for the header in this case. This option available only for home page. On sub-pages skin color will be used.

  • Custom Logo upload your logo image and retina logo here. Usual logo will appear on header, small logo will be displayed on sticky header and in header on mobile devices

  • Top Links this subsection allows to make top links looking exactly as on default magento (simple list). If the default view mode of links is chosen then you can disable any of those link

  • Top Cart options it is here you can display or hide next elements of top cart: product counter, top cart price, and text of top cart. At least one element should be active, otherwise all the top cart elements will be shown. Under this menu you also can enable option that will show top cart content on mouse over

  • Footer in this subsection of theme options you can activate custom footer and indicate which exactly static block will be used as footer. To create footer with the same content as on demo version enable custom footer in theme options and paste static block's ID with footer content. Content for that static block you will find in Sources/Static Content/Blocks/bizarre_footer.txt

  • Newsletter here you can choose background for the newsletter block which is shown by default on home page above the footer block

  • Product Listing here you can choose position of sidebar (left or right), set quantity of products in 1 row (from 2 to 8) in grid view of category, and enable or disable such product toolbar elements as: Quick view, Product Name, Product Price, "Add to cart", "compare products" wishlist, "mail a friend", more views buttons. All these options except product names alignment can be applied globally or for specific category (Category Edit page -> tab Meigee/Layout)

  • Product Images here you can choose type of product images displaying: "As is" option will display product images without any transition: the width is set in .phtml templates, the height is calculated automatically. "Keep the same height" option will display your products with the same height and keeping correct aspect ratio (spaces will be added if required). "Width and height" will allow you to choose one of predefined aspect ratios or set your own one. Be aware all these options will be applied to images of all products

  • Language Switcher allows you to choose display mode of language's choice and activate language label. To read more about creating multi language store you can in this thread: Multilingual online shop We included only a few flag icons, so if you do not see your flag icon you have to upload it to skin/frontend/bizarre/default/images/lang/ folder of your magento. Full list of flags with all available resolutions you can find here: flag images collection - click on necessary image, choose correct format and size and download it to the folder above

  • Currency Switcher allows you to choose display mode of currency's choice and activate currency label. To activate currency choice you have to add additional currency as allowed in System -> Configuration -> General -> Currency Setup and set currency rates in System -> Currency Setup. Same here you can choose icon symbol.

  • Fancybox this options enables customer to open product image in full size window. Available for home page, product listing, product page and for product which will be shown in footer. Do not forget to enable fancybox icon in product widgets if you decide to use that option for products

  • Product Hover Effect will display another image on mouse over. To get this works you should enable this feature in theme options and also you will need to indicate hover image on product edit page in tab "Images". Add word "hover" to the Label field of the required image. Hover image and base image should be different. Hover image should NOT be excluded from the product gallery. Bizarre theme has 2 types of hover: default hover and hover with scale effect.

  • Product Labels this menu enables displaying of labels "New", "Sale" and "Only X left" on product listings and product pages and same here you can manage labels' order and set 1 of 4 labels' view. This subsection also allow you to disable sale percentage on "Sale" label, word "sale" will be displayed in that case.
    Label "New" will be shown in case if the product is marked as new on product edit page in field "Set Product as New from Date".
    Label "Sale" will be shown if the product is marked with special price on product edit page in tab "Prices".
    Label "Only X left" will be shown if the stock level is equal or below the threshold specified in the admin panel. The threshold can be set in Only X left Threshold field in System -> Configuration -> Inventory". Only X left" label will replace "Sale" label

  • Login Ajax by enabling this function you give you customer opportunity to login on any page without annoying page reloading

  • Popup content here you can enable popup with static content which will be shown customers when they visit your store. You can add any static content to that popup. To create popup with the same content as on demo version enable popup in theme options and paste static block's ID with popup content. Content for that static block you will find in Sources/Static Content/Blocks/bizarre_popup_content.txt. Also here you can choose where that popup should be displayed (only on home page or on all pages), set width and height of the popup window and decide if you want to display popup on mobile and tablet devices. If quantity of categories in 1 row will be less than set value then they will fill all the space of megamenu. Example: if set 3 subcategories in 1 row and you have 5 subcategories, then first menu row will have 3 subcategories and second row will be divided in 2 equal parts for remaining subcategories. Known problems with a category page

Category page doesn't display layered navigation

Solution. Navigate to Catalog -> Manage Categories -> Category edit page -> Display Settings tab and set “Yes” in "Is Ancor" dropdown list

Categories is not displaying in navigation menu

Solution 1. Check your category tree. All categories should be under the Root categories
Solution 2. Check your category availability: Go to Catalog -> Manage Categories -> Category edit page -> General information and set "Yes" in "Is Active" dropdown list
Solution 3. Check if category included in navigation menu: Go to Catalog -> Manage Categories -> Category edit page -> General information and set "Yes" in "Include in Navigation Menu" dropdown list
Solution 4. Reindex all the data (System -> Index Management) and flush all the cache (System - Cache Management)

Products are not displayed on product listing

Solution 1. The products must be Visible in Catalog (Catalog -> Manage products -> Product edit page -> General tab -> Status -> Enabled)
Solution 2. The products must be Enabled (Catalog -> Manage products -> Product edit page -> General vertical tab -> Visibility -> Catalog, Search/Catalog)
Solution 3. Product must have a stock Quantity (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Qty -> at least 1)
Solution 4. The product must be set to In Stock (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Stock Availability -> In stock)
Solution 5. If the product is set not to track stock, it still has to have a stock Quantity and be set to In Stock
Solution 6. The product must be assigned to the target Category (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Categories vertical tab)
Solution 7. If using multi-website mode (or if you imported the products through Data Flow), the products must be assigned to the target Website (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Websites vertical tab)

    • Open skin/frontend/bizarre/default/css/custom.css file if you are using default theme
    • Open skin/frontend/skinX/default/css/custom.css file if you are using subthemes
  3. Add found selector with display: none rule inside the section
    • @media only screen and (min-width: 978px) and (max-width: 1280px) - if you want to hide element on devices with screen resolution from 978 up to 1280 px (tablet version landscape orientation)
    • @media only screen and (min-width: 768px) and (max-width: 977px) - if you want to hide element on devices with screen resolution from 768 up to 977 px (tablet version portrait orientation)
    • @media only screen and (min-width: 480px) and (max-width: 767px) - if you want to hide element on devices with screen resolution from 480px up to 767px (mobile version landscape orientation)
    • @media only screen and (max-width: 767px) - if you want to hide element on devices with screen resolution up to 767px (mobile version portrait orientation)
So, if you want to hide home banners on the mobile device of portait orientation, your end code in custom.css file should look like the code below: 
@media only screen and (max-width: 767px) {
.home-banners {display:none}
}

960 Grid System



The 960 Grid System is an effort to streamline web development workflow by providing commonly used dimensions, based on a width of 1252 pixels. We are using 12 columns grid in Bizarre theme which is divided into portions that are 86 pixels wide. Next names of classes are used for wrapping elements:

  • .grid_1 - width 86px
  • .grid_2 - width 192px
  • .grid_3 - width 298px
  • .grid_4 - width 404px
  • .grid_5 - width 510px
  • .grid_6 - width 616px
  • .grid_7 - width 722px
  • .grid_8 - width 828px
  • .grid_9 - width 934px
  • .grid_10 - width 1040px
  • .grid_11 - width 1146px
  • .grid_12 - width 1252px
All that classes have 10px left and right margins. Additional classes alpha and omega remove those margins accordingly: alpha removes left 10px margin, omega - right 10px margin. By default we use alpha for first element in grid and omega for the last element. Below is simple example how you can row with 4 equal section in which you can insert any information you need:

</div>

	<div class="grid_12">

	</div>
</div>

Customization of theme and using of custom subthemes



Magento uses parent/child theming concept so our theme is based on default magento theme. It allows to customize any parts of a theme with ease.

You should never modify magento and our theme default template files. This may cause lot's of problems in updating magento core or our theme to the latest version.

How to customize the theme

There are two correct ways how you should customize our theme. Choose it based on your needs

  1. You need to customize css rules only. In this case you should write your own css rules in skin/frontend/bizarre/default/css/custom.css
    Since this file is loaded after any other css files all css rules written in it will override any others
  2. You need to customize .js or .phtml files. In this case the only correct way is creating your own subtheme.

How to create subthemes

Creating a subtheme is a correct way of customization our theme. It allows you to update our theme to the latest version with ease. All you need is just upload newest version of the theme, implement changes in overridden files in your own subtheme. You can create as many subthemes as you want. You can create specific subthemes for categories or cms pages, etc.

Default theme folders structure.
app/design/frontend/bizarre/default/
Template files are separated into following folders:
layout - contains XML files which define pages structure
template - contains template .phtml files
locale - contains CSV files with translations

skin/frontend/bizarre/default/
Template files are separated into following folders:
css - contains CSS files
images - contains images
js - directory contains JavaScript files

Overriding of our extensions is not recommended and this guide doesn't cover it. You may have strong php/Zend knowledges to customize any extensions including ours.

To create your own subtheme you need to create two following folders:
app/design/frontend/bizarre/subtheme/

Integration of New Extensions with the Theme



There is the only one theme which is fully compatible with any extensions. Default magento theme.
We tried to make this theme as much compatible with any extensions as possible but it's impossible to get it done for sure due to huge amount of available extensions. So please notice this theme may not be compatible with some extensions.

Why the theme can't be compatible with any extensions?

The answer is pretty simple: If this theme and an extension use the same template files - they aren't compatible

Solution?

In most of cases the solution is pretty easy too:
Move template files of an extension to the theme folder. This theme is an interface, like base or default one, so if an extension has folder under base/default/ or default/default/ move all of them to bizarre/default/

But sometimes you also need to edit some template files of an extension or/and of the theme ones to get it working. We strongly do not recommend doing this if you are not familiar to magento, hire professional developers to get it done instead. Please refer to Custom Development chapter of this guide for more information.

Authors of the 3-rd party extensions should provide detailed instruction about how to integrate the extension with custom themes because default magento theme is used only on magento demo, on real stores people use custom themes for sure.

    Solution 1: Check if brand logo image is created and exist in media/brands folder of your magento

    Solution 1: Mark product as “New” or Sale on product edit page
    Solution 2: Make sure that you have enough new products to show. Usually it should be not less than 7 products marked as new/sale
    Solution 3: Reindex all the data (System-> Index Management) and flush the magento cache (System->Cache management)

    Solution: Make sure you have unzipped extension files correct way. All folders and files of this extension must be in appropriate directories of your magento. If you see aw_blog folder in the root of your magento you uploaded/unzipped it wrong way.
    Blog link doesn't work in different store view (i.e. language)
    Solution: Set path to the blog considering language path for each store (i.e. /english/blog) in "Custom link" field of blog category

    Category page doesn't display layered navigation
    Solution: Navigate to Catalog -> Manage Categories -> Category edit page -> Display Settings tab and set “Yes” in "Is Ancor" dropdown list
    Categories is not displaying in navigation menu
    Solution 1: Check your category tree. All categories should be under the Root categories
    Solution 2: Check your category availability: Go to Catalog -> Manage Categories -> Category edit page -> General information and set "Yes" in "Is Active" dropdown list
    Solution 3: Check if category included in navigation menu: Go to Catalog -> Manage Categories -> Category edit page -> General information and set "Yes" in "Include in Navigation Menu" dropdown list.
    Solution 4: Reindex all the data (System -> Index Management) and flush all the cache (System - Cache Management)
    Products are not displayed on product listing
    Solution 1. The products must be Visible in Catalog (Catalog -> Manage products -> Product edit page -> General tab -> Status -> Enabled)

    Solution 3. Product must have a stock Quantity (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Qty -> at least 1)
    Solution 4. The product must be set to In Stock (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Stock Availability -> In stock)
    Solution 5. If the product is set not to track stock, it still has to have a stock Quantity and be set to In Stock
    Solution 6. The product must be assigned to the target Category (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Categories vertical tab)
    Solution 7. If using multi-website mode (or if you imported the products through Data Flow), the products must be assigned to the target Website (Catalog -> Manage products -> Product edit page -> Inventory vertical tab -> Websites vertical tab)
    Solution 8. You must refresh your Cache (System -> Cache management) and reindex all the data (System -> Index Management)

Self-Debugging



Since we are providing only theme related support please make sure that your issue is definitely caused by theme before sending a request to support team. How to debug that:

  1. Disable all the cache (System -> Cache management) and activate default theme (System -> Configuration -> Design -> Package -> Set as default) Save changes and check if the problems remains. If so, the issue is not theme related and you need to dig your magento or custom modules settings in order to find the reason of issue.
  2. Disable all the modules which were previously installed (just rename file of module in this folder "app/etc/modules" Make sure that cache is disabled in your admin before renaming modules). If the issue disappear after you renamed all your custom modules it will mean that one of your module breaks theme behaviour. Revert all the changes back and start to rename one by one your modules files in order to know which exactly module breaks theme behaviour
  3. Just check same issue on demo version. If the demo version doesn't show same error, it means that you have error in your store only, so you have to explore the reason of issue in your store. Refer to the points above in order to do that.
  4. Open console of Firebug or similar debug tools and update page. If you see many errors like "theme_feature is not a function" it means that you included more than jQuery library. Refer to the point 2 in order to know which exactly module requires additional library.

First of all make sure you have flagged this option on your downloads page to get notified by email if the theme is updated

After getting an email with note that the theme was updated you need to download the latest version on your downloads page by yourself. The newest theme always comes with folder "Changes" where we put .diff files with all changes we have made.
